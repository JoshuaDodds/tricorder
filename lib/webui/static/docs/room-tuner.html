Room tuner calibration

The room tuner utility (room_tuner.py) captures short windows of audio from your configured device, tracks the ambient noise floor, and recommends new segmenter.rms_threshold values. The dashboard’s Calibrate noise profile shortcut opens this guide so you can launch the tool without leaving your browser session.

Requires shell access. The tuner still runs from the command line. Use SSH or a local terminal on the recorder and execute the commands below.

Quick start

  1. SSH into the recorder: ssh pi@tricorder.local
  2. Start the tuner with your audio device and current gain: 
    cd /apps/tricorder
./room_tuner.py --device "$(yq '.audio.device' config.yaml)" --gain $(yq '.audio.gain' config.yaml)
  3. Let it run for 30–60 seconds while the room is quiet to capture the baseline noise floor.
  4. Note the suggested RMS_THRESH printed in the console. Update segmenter.rms_threshold or enable the automatic noise profile quick action in the dashboard to apply it later.

Optional flags

  • --aggr <0-3> — match the WebRTC VAD aggressiveness configured in the dashboard.
  • --margin <float> — scale the noise floor when computing the recommended threshold.
  • --csv room_tune.csv — log interval stats for offline analysis.

When you enable the calibration shortcuts in the recorder settings modal the dashboard reminds you to rerun the tuner as part of regular maintenance. The Calibrate noise profile button from the modal simply returns to this page so the steps are always one click away.
